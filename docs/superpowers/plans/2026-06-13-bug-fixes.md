## Task 4: Solve-token — защита от устаревших BFS-ответов

**Проблема:** даже с проверкой стейджа (Task 3) остаётся теоретический сценарий: BFS завершился, его сообщение в очереди, пользователь нажал Cancel, resetProgress сбросил всё, потом пользователь сразу нажал кнопку сложности и попал на config stage — и тут прилетает старый ответ. Task 3 защищает от перехода, но не от перезаписи `state.solution` и `state.cachedSolution`.

## Task 6: Валидация `deps` и структуры при импорте

**Проблема:** `applyImportedConfig` проверяет каждую плашку по отдельности, но не валидирует:
- поля внутри `deps` (`targetId`, `direction`, `steps`)
- что все плашки имеют одинаковый `positions`
- что `id` плашек образуют строго последовательность `1..N` (BFS использует `newPos[plateId - 1]`, что ломается при пропусках)

**Files:** `index.html:1194`

- [ ] Заменить функцию `applyImportedConfig` целиком:

```js
function applyImportedConfig(text) {
  let plates;
  try { plates = JSON.parse(text); } catch {
    showToast('Невалидный конфиг', 'error'); return;
  }
  if (!Array.isArray(plates) || plates.length < 2 || plates.length > 8) {
    showToast('Невалидный конфиг', 'error'); return;
  }

  const positions0 = plates[0]?.positions;
  // IDs must be exactly 1..N in some order
  const sortedIds = plates.map(p => p.id).slice().sort((a, b) => a - b);
  if (!sortedIds.every((id, i) => id === i + 1)) {
    showToast('Невалидный конфиг', 'error'); return;
  }

  for (const p of plates) {
    if (typeof p.id !== 'number' ||
        typeof p.positions !== 'number' || p.positions % 2 === 0 || p.positions < 3 ||
        p.positions !== positions0 ||
        typeof p.currentPos !== 'number' || p.currentPos < 1 || p.currentPos > p.positions ||
        !Array.isArray(p.deps)) {
      showToast('Невалидный конфиг', 'error'); return;
    }
    for (const d of p.deps) {
      if (typeof d.targetId !== 'number' ||
          !Number.isInteger(d.targetId) ||
          d.targetId < 1 || d.targetId > plates.length ||
          d.targetId === p.id ||
          (d.direction !== 'same' && d.direction !== 'opposite') ||
          typeof d.steps !== 'number' ||
          !Number.isInteger(d.steps) || d.steps < 1) {
        showToast('Невалидный конфиг', 'error'); return;
      }
    }
  }

  state.plates = plates;
  state.activePlate = 1;
  state.cachedSolution = null;
  document.getElementById('val-plates').textContent = plates.length;
  document.getElementById('val-positions').textContent = plates[0].positions;
  renderMatrix();
  buildScene('scene-config-inner', state.plates);
  updateScene('scene-config-inner', state.plates, state.activePlate);
  document.getElementById('btn-plates-dec').disabled = plates.length <= 2;
  document.getElementById('btn-plates-inc').disabled = plates.length >= 8;
  document.getElementById('btn-pos-dec').disabled = plates[0].positions <= 3;
  showToast('Конфиг применён');
}
```

- [ ] Проверить вручную — корректный JSON принимается:

Сгенерировать конфиг через Ctrl+C, затем вставить через Ctrl+V — должен применяться без ошибок.

- [ ] Проверить вручную — невалидный JSON отклоняется:

Попробовать вставить `[{"id":2,"positions":7,"currentPos":3,"deps":[]}]` (id не с 1) — должен показать тост «Невалидный конфиг».

Попробовать `[{"id":1,"positions":7,"currentPos":3,"deps":[{"targetId":1,"direction":"same","steps":1}]}]` (самозависимость) — должен отклонить.

Попробовать `[{"id":1,"positions":7,"currentPos":3,"deps":[{"targetId":2,"direction":"bad","steps":1}]},{"id":2,"positions":7,"currentPos":4,"deps":[]}]` (невалидный direction) — должен отклонить.
